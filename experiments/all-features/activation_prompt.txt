## 执行阶段

### 阶段 1：启动与调度
- 扫描当前仓库
- 说明为什么要用 planner / devops persistent teammate
- 说明为什么 knowledge / code_reviewer 更适合 delegate

### 阶段 2：任务板
- 生成一组覆盖架构、权限、观测、持久化、实验验证的任务
- 让至少 1 个 teammate claim task

### 阶段 3：专项执行
- planner teammate 推进任务板
- devops teammate 做 runtime / dashboard / metrics 检查
- knowledge delegate 做 RAG 专项
- code_reviewer delegate 做代码专项
- 如果可见 `mcp_repo_probe`，调用并记录

### 阶段 4：归档
- 把所有关键结论与来源建立索引
- 明确区分：
  - 已验证事实
  - 基于代码推断
  - 失败/阻塞
